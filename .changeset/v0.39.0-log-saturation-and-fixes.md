## 0.39.0 — Log-saturation scoring, fix correctness, deprecation cleanup

### Scoring

- **Log-saturation category scoring** — `metrics.ts` now uses `min(100, log10(1 + points/500) / log10(11) * 100)` instead of `(points / componentCount) * 100`. Per-category scores are now comparable across project sizes, the divide-by-zero on `componentCount === 0` is gone, and the curve matches the saturation shape already used by `aiSlopScore`.
- **Louvain standard-formula fix** — modularity Q is now `Σ_c [σ_in / (2m) − (Σ_tot)² / (4m²)]` (the canonical Newman–Girvan form) instead of an off-by-2 divisor.

### Correctness

- **`dead/unused-import`** correctly handles aliased imports — `import { useState as myState }` is now read as importing `useState`, not `myState`.
- **`dead/dead-branch`** no longer fires on the canonical `while (true) { break; }` event-loop pattern.
- **`shadcn-registry`** refresh no longer fetches the dead upstream URL (it 404s); the bundled snapshot at `src/data/shadcn-registry.json` is now authoritative.

### Deprecation

- 3 deprecated MCP tools (`slop_governance`, `slop_architecture_score`, `slop_business_logic_score`) are removed in this version. All three were strict subsets of `slop_suggest`. Their fields (`repositoryHealth`, etc.) are accessible via `slop_suggest` directly.

### Tests

- 5 metrics tests + 3 louvain tests + 1 dead-branch test + the unused-import positive test were rewritten to express the v0.39.0 contracts as explicit invariants. No test was deleted.
