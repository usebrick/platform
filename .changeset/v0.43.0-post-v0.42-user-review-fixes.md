## 0.43.0 — Post-v0.42.0 user-review fixes

A user walked through the v0.42.0 build as a first-time user and
found ~25 small UX bugs. All fixed in single-purpose commits.
Purely bugfix; no npm-publish-relevant rule changes. **Recommended:
cherry-pick back into your codebase to keep the headline output
consistent.**

### Headline / brief output
- Corrected verdict band for aiSlopScore in `--brief` and
  `formatVerdict()` — used to show "concerning" for score=25 because
  the higher-is-better `scoreBand()` was applied to a
  lower-is-better score. Now `slopScoreBand()` (0=no slop, 25:low,
  50:high, 70:saturated).
- All hardcoded `30` thresholds (brief gate, why-failing headline,
  exit-code error, JSON field, 4-score footer, next-step hint) now
  read `report.thresholds.meanSlop`. The slopbrick repo's stricter
  `meanSlop=15` now reads as "≤ 15 -> fail" everywhere (was
  "≤ 30 -> pass" — wrong).
- `--why-failing` headline and 4-score footer normalized to use
  real `meanSlop` rather than hardcoded 30/70.

### Help / command output
- `slopbrick --help` now includes the full v0.42.0 subcommand list
  (was a self-deprecating subset).
- `--no-increase` help text: "if slopIndex increased" → "if
  aiSlopScore increased (lower = cleaner since v0.21)".
- `slopbrick ci --max-slop` help: same direction wording.
- `slopbrick --version` is reachable through the bin script
  (was previously behind a bin error).
- `slopbrick scan` "no source files" warning now suggests
  `--workspace` (was: "Adjust --include", which is the wrong fix
  for monorepo users).
- `slopbrick explain` with no arg now suggests `slopbrick rules`
  instead of Commander's bare "missing required argument".
- `slopbrick doctor` (a) no longer claims "31 rules configured
  in your config file" when no config exists; (b) hints at
  `--workspace` in the "No source files" warning; (c) prints
  "Node X, slopbrick Y" on the first line so bug reports have
  the tool version baked in.

### Patterns / data output
- `slopbrick patterns` now uses "✓ none" (not "✓ clean") when
  count is 0, so a repo with 0 modals doesn't get a misleading
  "all clean" verdict.
- `slopbrick --incremental` cache writeback now actually runs
  (`saveCache()` was missing). 2nd scan skips unchanged files.

### Error wording
- Exit-code error upgraded from "N threshold(s) failed. See details
  above." to "N threshold(s) failed: meanSlop (score X > Y)" with
  the actual score and limit named.
- Bundled-snapshot message no longer says "404" (was alarming
  about a non-issue); reads "built-in; no network fetch".

### Scoring direction consistency
- Trend subcommand, flywheel summary, maintenance-cost axis, pretty
  secondary-view footer, README, AGENTS.md, scoring-explained.md,
  architecture.md, repository-structure.md, perf test label: all
  renamed `Slop Index` → `AI Slop Score (lower = cleaner since
  v0.21)`.
- packages.json `engines.node` ">=18" → ">=20" to match doctor's
  authoritative Node check.

### JSON output
- New optional `failedThresholds` field on ProjectReport.
  E.g. `["meanSlop"]`. Additive; v0.42 and earlier consumers
  ignore the field.

### Internal docs
- Internal JSDoc comments on `MaintenanceAxes.aiSlopScore`,
  `RepositoryHealthInputs.aiSlopScore`, and the `ProjectReport`
  comment block — all previously said "higher = better" (the
  v0.15-era reading); now correctly document v0.21+ raw amount
  with the lower = better direction.
- All MCP tool descriptions (`slop_scan_file`, `slop_suggest`,
  `slop_find_similar`, `slop_check_constitution`,
  `slop_explain_rule`) updated to enumerate the actual returned
  field names; the previous descriptions referenced fields the
  implementation never emitted.

### Bin script
- `bin/slopbrick.js` now wraps the entry-point IIFE in try/catch
  + process handlers for `unhandledRejection` / `uncaughtException`.
  Top-level crashes exit 1 with a one-line stderr message
  `slopbrick: <error class> — <message>` instead of an unhandled
  rejection warning + stack trace.

### Tests
- New `tests/cli/threshold.test.ts` — 5 cases for `failedThresholds()`,
  `failedThresholdCount()`, multi-gate behavior, count consistency.
- New `tests/cli/drift.test.ts` — 5 cases asserting actual threshold
  wording after the rename.
- New `tests/report/html.test.ts` case asserting the HTML output's
  threshold row uses `report.thresholds.meanSlop`.
- Updated `tests/cli.test.ts` — assertion now reads the new error
  wording.
- Updated `tests/integration/watch-mode.test.ts` — regex now
  matches the post-v0.21 surface.

### Notes

- **`engines.node`** bumped from `>=18` to `>=20`. Users on Node 18
  will see npm install warn + `slopbrick doctor` fail.
- **`previousSlopIndex`** field name kept; the value is now the
  previous `aiSlopScore` (since v0.21.0). Pre-1.0 field name change
  is the doc, not the wire format.
- **No rule additions, removals, or rank changes.** Pure
  output/wording/UX fixes. `signal-strength.json` is unchanged.
- Self-scan after the arc: `aiSlopScore 25` (slopbrick repo's
  strict `meanSlop: 15`), matching v0.42.0 baseline.
