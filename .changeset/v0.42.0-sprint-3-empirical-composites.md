## 0.42.0 — Sprint 3 (empirical composites + AGENTS.md refresh + PR Action)

### §3b — Empirical composite rules

- `slopbrick composite discover` runs the 5-step algorithm (adaptive support, NPMI + Fisher's exact, single-linkage clustering, F1 sweep × recall/FP ≥ 1.5×, sha1-derived composite id) over `.slopbrick/flywheel/scans.jsonl` and writes `composites.json`.
- `slopbrick composite enable <id>` writes the chosen composite into `slopbrick.config.mjs#compositeRules`.
- New `CompositeRule<Context>` type — structurally parallel to `Rule<Context>`. `defaultOff: true` until user opts in.
- New hand-curated seed (`composite/ai-fingerprint-cluster`) shipped when cold-starting, dropped once auto-discovery dominates.
- Engine wiring: `facts.compositeFireSet` carries the fired-rule-id set per file; the engine's existing pass over the registry fires composites like any other rule.
- F6 fix (carry): deleted dead 2-arg `register(id, factory)` overload on `RuleRegistry`.

### §3a — AGENTS.md auto-refresh

- `<!-- slopbrick:begin:v3 --> / <!-- slopbrick:end:v3 -->` markers delimit a managed block in AGENTS.md / CLAUDE.md.
- `safeRewrite` semantics: both markers → replace inner; neither → fail closed (no clobber); one → warn + skip.
- Atomic write via `path.tmp → path` rename.
- Opt-in via `--refresh-snippets` CLI flag or `autoRefreshSnippets: true` in `slopbrick.config.mjs`. Default off.
- Hooked in `persistRun.ts` after `recordTelemetry`, so a partial scan failure doesn't surface stale data. Wrapped in try/catch — failure here is not a scan failure.

### §3c — PR review surface

- `slopbrick pr --diff <file>` accepts a unified-diff file path (e.g. `git diff base...head > /tmp/pr.diff`). Parses `+++ b/path` headers; scores the on-disk current contents of each changed path. No git binary at runtime.
- New GitHub Action at `.github/workflows/slopbrick-review.yml` posts a single `<!-- slopbrick-review -->`-marked comment per PR; deletes prior slopbrick comments to keep re-runs idempotent.
- Permissions narrowed: `pull-requests: write`, `contents: read`, `actions: read`. Single human gate via the `review` environment.
