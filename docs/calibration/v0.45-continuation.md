# v0.45 calibration continuation

The authoritative execution plan is the package-level
[v0.45.0 continuation plan](../../packages/slopbrick/docs/calibration/v0.45.0-continuation-plan.md).
This note records only the current ordering; it does not mark any gate passed.

## Current dependency sequence

1. **Gate 3 — lossless pipeline:** finish versioned calibration artifacts,
   stable per-file observations, fail-closed accounting, deterministic
   selection/run metadata, retry/bisection, coverage checks, and schema-backed
   machine output.
2. **Gate 4 — provenance corpus:** create and review the versioned corpus
   manifest; quarantine unproven positives; record pinned origin, license,
   label evidence, file hash, family/duplicate cluster, stratum, and split
   eligibility; then derive file lists only from that manifest.
3. **Gate 3 + Gate 4 integration:** use the reviewed manifest as the only
   selection input. A run must account for every selected file exactly once and
   fail closed on stale, missing, duplicate, corrupt, or mismatched records.
4. **Gate 5 onward:** run smoke, canary, and full calibration only after the
   preceding gates have their required evidence. ML research remains downstream
   of valid per-file data and leakage-safe splits.
