# v0.15.0 — Merged Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

> **This is a merged plan.** Detailed steps for the architectural refactor (Phases 1–4) are in `2026-06-26-architectural-refactor.md`. This document is the single source of truth for sequencing, file maps, acceptance criteria, and the new phases (0, 5, 6) that the original plan didn't cover.

**Goal:** Ship slopbrick v0.15.0 — a hard-break rebrand to "Repository Structure Platform" + a typed verdict contract + an extracted engine package + a typed schema codegen + website hardening + engine/UI taxonomy split + multi-dimensional scoring.

**Architecture:**
- **Phase 0 (Rebrand)**: rename all `memory*` → `structure*` files, types, exports; bump `MEMORY_SCHEMA_VERSION='2'` → `STRUCTURE_SCHEMA_VERSION='3'`; bump slopbrick 0.14.5 → 0.15.0; rewrite CHANGELOG.
- **Phase 1 (P0)**: verdict taxonomy becomes a typed contract via `packages/core/src/verdicts.ts`; Zod schema validates `signal-strength.json` at load time; property-based tests replace exact counts; docs land.
- **Phase 2 (P1)**: website hardening (a11y, WebGL cleanup, low-power, axe-core CI) + schema codegen (`packages/core/scripts/codegen-types.ts` reads JSON Schemas, writes TypeScript).
- **Phase 3 (P1)**: `packages/engine/` extracted as a new workspace package. Pure functions, no I/O. CLI becomes a thin wrapper.
- **Phase 4 (P2)**: per-brick shader jitter + LCP-swap (WebGL waits for LCP before initializing) + preload brick-pattern SVG.
- **Phase 5 (new)**: positioning copy on the website + all docs (Hero, Compare, CTA, Footer, tagline, title, README, AGENTS, CHANGELOG).
- **Phase 6 (new)**: engine/UI taxonomy seam — engine keeps 6 verdicts, UI exposes 3 buckets (AI Findings / Engineering Hygiene / Suppressed). Multi-score: replace `slopIndex` with `aiQuality` / `engineeringHygiene` / `security` / `repositoryHealth`.

**Tech Stack:** TypeScript 5.6, Node 18+, pnpm 9, Vitest 2, Zod 3, Playwright 1.4x + axe-core 4.x, json-schema-to-typescript, Astro 4.16, GSAP 3.12, Lenis 1.1.

## Critical Sequencing Constraints

These must be respected; violation causes a multi-day rework:

1. **R.1–R.9 (rebrand renames) MUST run BEFORE Phase 1 tasks A.1, A.2, A.3.** Phase 1 creates new files (`verdicts.ts`, `signal-strength-schema.ts`) that import from the renamed paths. If R.1–R.9 hasn't run, those new files would import `memory` paths and immediately need re-renaming.
2. **R.10 (schema version bump) MUST run before Phase 6 (multi-score) and before any new schema field is added.** The new `STRUCTURE_SCHEMA_VERSION='3'` is what gates consumers' migration.
3. **B.4 (move memory persistence to engine/) MUST run after R.4 (rename engine/memory.ts → engine/structure.ts).** B.4 moves a file that R.4 has already renamed.
4. **Phase 6 (multi-score) MUST run after Phase 1 (verdict type safety).** Phase 6 uses the `Verdict` enum and `bucketForVerdict()` function that Phase 1's new files provide.
5. **Phase 5 (positioning copy) MUST run last** (after all type/function renames are done) so the copy refers to the final type names, not the in-flight ones.

The `memory*` names are replaced with `structure*` everywhere. **Slop Score is replaced by 3 independent scores** (AI Quality / Engineering Hygiene / Security + a Repository Health composite).

```markdown
### Breaking changes

- `MEMORY_SCHEMA_VERSION` (value 2) is replaced by `STRUCTURE_SCHEMA_VERSION` (value 3). The schema field `slopIndex` is replaced by `aiQuality` / `engineeringHygiene` / `security` / `repositoryHealth`.
- The on-disk artifact `.slopbrick/memory.md` is renamed to `.slopbrick/structure.md`.
- Types: `MemoryFile` → `StructureFile`; `MemoryCategory` → `StructureCategory`; `MemoryPattern` → `StructurePattern`.
- Functions: `loadMemory` / `saveMemory` → `loadStructure` / `saveStructure`.
- MCP tool `slop_suggest_with_memory` → `slop_suggest_with_structure` (the tool identifier changes).
- The verdict enum is unchanged (6 values) but the user-facing report now exposes a 3-bucket taxonomy (AI Findings / Engineering Hygiene / Suppressed) instead of the 6 verdict names. The engine/UI seam is a new `bucketForVerdict()` function in `packages/slopbrick/src/report/buckets.ts`.
- New `packages/engine/` workspace package — the pure scanning logic is extracted from `slopbrick/src/engine/`.

### Migration

There is no automatic migration. Update any code that reads `.slopbrick/memory.md` to read `.slopbrick/structure.md`. Update any consumers of the `slop_suggest_with_memory` MCP tool to call `slop_suggest_with_structure` instead.

### Added
- `Verdict` enum, `isDefaultOff()`, and `VERDICTS` are now exported from `packages/core/src/verdicts.ts` as a single source of truth.
- Zod schema `signalStrengthSchema` validates the calibration data at load time.
- Schema codegen: `packages/core/scripts/codegen-types.ts` reads `schemas/v1/*.json` and writes TypeScript types to `packages/core/src/generated/`. CI fails if schemas and types are out of sync.
- WebGL cleanup: `brick-shader.ts` calls `WEBGL_lose_context?.loseContext()` on unmount.
- Tool cards are keyboard-accessible (button role + Enter/Space + focus-visible).
- Skip-to-content link.
- axe-core via Playwright in CI for the website.
- `LowPowerDetector` skips WebGL on devices with `deviceMemory < 4` / `hardwareConcurrency < 4` / `prefers-reduced-motion`.
- Per-brick jitter in the WebGL shader.
- LCP-swap: WebGL canvas waits for `largest-contentful-paint` before initializing.
- `bucketForVerdict()` — the engine/UI taxonomy seam.
- Multi-score: `aiQuality` / `engineeringHygiene` / `security` / `repositoryHealth`.
```

```ts
import type { Verdict } from '@usebrick/core';

/**
 * v0.15.0+: The user-facing 3-bucket taxonomy. Maps engine verdicts
 * (6 values, used for LR math and calibration) to UI buckets (3 values,
 * used in the report and JSON output).
 *
 * Per the user's design intent ("INVERTED is basically HYGIENE"), INVERTED
 * rules are in the same UI bucket as HYGIENE rules. The engine still
 * distinguishes them (INVERTED has LR < 1, HYGIENE has LR ≈ 1).
 */
export type Bucket = 'ai' | 'hygiene' | 'suppressed';

export function bucketForVerdict(verdict: Verdict): Bucket {
  switch (verdict) {
    case 'USEFUL':
    case 'OK':
      return 'ai';
    case 'HYGIENE':
    case 'INVERTED':
      return 'hygiene';
    case 'NOISY':
    case 'DORMANT':
      return 'suppressed';
  }
}

/** Convenience: count rules per bucket. */
export function bucketDistribution(verdicts: Verdict[]): Record<Bucket, number> {
  const dist: Record<Bucket, number> = { ai: 0, hygiene: 0, suppressed: 0 };
  for (const v of verdicts) {
    dist[bucketForVerdict(v)]++;
  }
  return dist;
}
```
