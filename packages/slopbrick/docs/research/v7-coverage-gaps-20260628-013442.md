## Where to look for new rules

The gap files cluster in the following patterns:

1. **Agent harness repos** (`vibe-coded/*`, `cline/*`, `continue/*`,
   `aider/*`, `claude-code*`) — these are AI frameworks whose own
   code is hand-written to be reliable, so they look "human" to our
   rules. New rules could target "agent code" specifically (e.g.,
   tool-call patterns, prompt templates, model selection).

2. **File types we don't cover well** — check the extension table.
   If a big bucket is a language we have no rules for, that's
   P0 work.

3. **Test files** — many gap files end in `.test.ts`, `.spec.ts`,
   `__tests__/`. AI-generated tests have their own signatures
   (predictable setup, fake placeholders, snapshot overuse).

4. **Type definition files** — `.d.ts`, `types.ts`, `interface.ts`.
   AI generates these in bulk; they have low entropy and few tokens,
   so most AI rules miss them.

5. **Config / build files** — `next.config.js`, `vite.config.ts`,
   `webpack.config.js`. Sparse, high-boilerplate, no logic to flag.
