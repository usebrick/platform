## v10.3 immutable release materialization — Task 3 plan correction (2026-07-11)

Task 3 RED/design review found that the original release plan's atomic-rename
language conflicted with its no-overwrite invariant: POSIX rename may replace
an existing digest target. Current official Node, POSIX, IANA, and RFC research
also required distinct raw-header inspection, explicit abort/deadline behavior,
complete per-hop address/peer validation, and constant redacted errors.

The release plan was corrected from SHA-256
`0115a2c73dcfd0a086c70b1e3fe6519558bf7c34ff4942d97d52c3dfa92c585d`
to `7a20d89c35dcea0d9255e21d3e9cdadf89404c229a7c2dcb474b6bbacac7cb41`.
Task 3 now uses Node built-ins only, publishes by same-directory
`link`-if-absent plus winner rehash, preserves invalid existing entries,
examines uncollapsed headers, disables connection reuse, validates every DNS
answer and the connected peer on every hop, and specifies 30-second idle and
five-minute whole-operation aborts. Package and lockfile changes remain owned
by Task 4's exact ZIP dependencies.

The targeted independent rereview in
`.superpowers/sdd/v103-release-task3-plan-rereview.md` returned APPROVE/READY;
its zero-context diff confirmed that Tasks 1-2 and 4-8 were otherwise
byte-unchanged. The admission plan remains frozen at
`233ede7262dd408318230f32ab4cc5d2103caf8b5662ae4fb228088531f70e75`.

Follow-on OWASP/SSRF review found that arbitrary RFC 6052 network-specific
NAT64 prefixes cannot be distinguished safely from native global IPv6 without
trusted-prefix or network-sandbox authority. It also found that a leaf-only
cache check does not prevent a writable-parent swap and that silently omitting
POSIX no-follow primitives would overstate native Windows safety. The plan was
therefore tightened first to IPv4-only acquisition at candidate SHA
`2fea604dd10c3a1bb91cd68ebd9ca9fdbb1f9901e8b609e099f204bca25b099e`
and then to the final reviewed SHA
`3c115b75b63544c8fa281fb32532be25ed5d0a71b0afad7571163829668e93c5`.
The final contract adds a pre-existing canonical POSIX cache root with a fully
trusted ancestor chain, fail-closed unsupported-platform behavior, a 5-GiB
absolute ceiling, explicit certificate/TLS/header/parser controls, and honest
cooperative filesystem-deadline wording. Native IPv6 and reparse-safe Windows
support now require later explicit designs rather than silent fallbacks.

Independent targeted verdicts are persisted at
`.superpowers/sdd/v103-release-task3-ipv4-plan-rereview.md` and
`.superpowers/sdd/v103-release-task3-cache-plan-rereview.md`; both returned
APPROVE/READY and confirmed Task 3-only plan scope.
