# usebrick.dev — marketing site

The brick-themed landing page for the slopbrick monorepo. Astro + Lenis
+ GSAP, with a WebGL fragment shader for the hero brick wall.

## Dev

```bash
# From the monorepo root:
pnpm --filter @usebrick/website dev
# → http://localhost:4321
```

## Build

```bash
pnpm --filter @usebrick/website build
# → packages/website/dist/
```

The `prebuild` script (`scripts/prebuild.ts`) reads
`../slopbrick/package.json` and writes `src/data/version.json` so the
install command + footer version stay in sync with the published
slopbrick release.

## Architecture

- **Astro 4** with `output: 'static'` for SSG
- **Astro Islands** for the few interactive pieces:
  - `BrickShader` (`client:load`) — WebGL canvas in the hero
  - `BreakOnHover` (`client:visible`) — click-to-break on the 4 tools
- **Lenis** for smooth scroll (initialized in the Base layout)
- **GSAP** for ScrollTrigger reveals, stat counter animation, click shake
- **Tabler Icons** for the small icon set (file, copy, etc.)

## Theme

The brick theme is defined by CSS custom properties in
`src/styles/global.css`. The brand is **terracotta on warm dark
mortar** — `#dc4a26` accent on `#1a0e08` background. All semantic
state colors (success, warning, danger) are warm tones that fit the
brick palette rather than the default red/green/blue.

## File structure

```
src/
├── pages/
│   └── index.astro          # the single landing page
├── layouts/
│   └── Base.astro           # shared <head>, font loading, Lenis + GSAP init
├── components/              # one .astro per section
├── scripts/                 # TS modules (shader, reveals, counters, copy)
├── styles/                  # global.css (tokens), theme.css, components.css
└── data/                    # version.json (generated by prebuild)
```

## Deploy

`.github/workflows/deploy.yml` runs on push to `main` (when files in
`packages/website/**` change). It builds the site, uploads the `dist/`
folder, and deploys to GitHub Pages.

If you need to deploy elsewhere (Cloudflare Pages, Vercel, Netlify),
swap the `actions/deploy-pages` step for the relevant provider's
action. The build step is identical.
